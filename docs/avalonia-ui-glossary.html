Angor app quick reference

Use these when describing issues in the Angor Avalonia app:

  • “The main content area” → OverlayBorder in ShellView
  • “The home section” → HomeSectionView
  • “The Find Projects card” → left column (column 0) in HomeSectionView
  • “The Launch a Project card” → right column (column 2) in HomeSectionView
  • “The card surface/background” → Border with Classes="Panel"
  • “The card content (icon, text, button)” → StackPanel inside the card
  • “The logo texture overlay” → Border with TiledLogoBrush
  • “The gap between the two cards” → middle column of Grid (*,24,*)
Tip: When reporting layout issues, say things like “the card surface” or “the StackPanel inside the Find Projects card” — that maps directly to the XAML structure.

Inspecting the running app

Run the app in Debug mode (F5), then press F12 to open Avalonia DevTools.

  • Logical / Visual Tree — Click a control to see it in the tree and inspect its properties
  • Layout tab — View and edit Margin, Padding, Border, size constraints
  • Ctrl+Shift+click — Inspect the control under the cursor
  • F8 — Screenshot the selected element

Use this to find which control has the wrong margins, then tell me the control type and property (e.g. “OverlayBorder.Panel Padding” or “ScrollableView ContentPadding”).
